# 06-03 Plan: Multi-Page Reconciliation

## Objective
- Introduce an explicit reconciliation pass that merges page-level extraction results into one package-level result set while preserving page + bbox evidence on the canonical PDF.

## Current State
- The extraction pipeline classifies from the first canonical PDF page only.
- Each page is extracted independently and persisted as a separate `Extractions` row.
- There is no package-level merge strategy for repeated fields, page conflicts, or multi-attachment evidence.
- The reviewer contract is now consistent around canonical PDFs, but the extracted data contract is still page-fragmented.

## Target State
- The pipeline extracts page-level results and then reconciles them into one coherent package-level extraction result.
- Reconciliation defines stable rules for first-wins, highest-confidence wins, repeated-field merge, and missing-field fill behavior.
- The reviewer can continue using page + bbox evidence because reconciled fields still point back to canonical PDF page evidence.
- The pipeline remains incremental: no schema rewrite or analytics redesign is required in this slice.

## Implementation Steps
1. Add a focused reconciliation service that accepts ordered page extraction results and returns one package-level result plus provenance metadata.
2. Update the extraction pipeline to run reconciliation after page extraction and persist a package-level reconciled record.
3. Keep page-level evidence available where needed, but make the reconciled record the primary extraction contract for downstream review.
4. Add tests covering multi-page conflicts, confidence-based merges, and missing-field completion across pages.

## Risks
- Compatibility risk: medium; downstream UI and queries may assume one row per page today.
- Migration risk: low to medium; existing extraction rows remain, but consumers may need to prefer reconciled output.
- Operational risk: low; reconciliation is in-process logic with limited runtime cost.

## Validation
- Tests to run:
  - `pytest tests/test_reconciliation_service.py tests/test_extraction_pipeline.py -q --basetemp=.pytest-lite-06-03`
- Manual verification:
  - ingest a multi-page document with fields split across pages
  - confirm one reconciled extraction appears for review
  - confirm selected fields still highlight the correct canonical PDF page and bbox
- Success criteria:
  - multi-page fields are merged into one coherent extraction result
  - confidence and provenance remain traceable to canonical PDF pages
  - reviewer behavior stays page + bbox based
