# 06-02 Plan: Durable Extraction Job Handoff

## Objective
- Introduce a persisted extraction job boundary between ingestion and extraction with idempotent enqueue, claim, and retry semantics.

## Current State
- The watcher performs ingestion and extraction inline in one process.
- The CLI batch processor infers pending work from `packages.status = INGESTED`.
- There is no persisted extraction job record, attempt counter, lease metadata, or retry policy.

## Target State
- Ingestion persists a package and enqueues a durable extraction job in SQLite.
- Workers claim jobs explicitly before running the extraction pipeline.
- Failed jobs requeue until a bounded retry limit is reached, then become terminally failed.
- Package statuses remain user-facing workflow state; job records track orchestration state.

## Implementation Steps
1. Add an `ExtractionJob` model and a service for idempotent enqueue, claim, completion, and failure transitions.
2. Update the watcher to stop after ingestion + enqueue, then move the source file to `processed`.
3. Update the CLI processing path to drain queued jobs rather than scanning package statuses directly.
4. Add focused tests for queue semantics, watcher handoff, and worker execution.

## Risks
- Compatibility risk: low to medium; operator behavior changes because extraction now requires the worker path after watch ingestion.
- Migration risk: low; package/extraction persistence stays intact and rollback can restore direct calls.
- Operational risk: medium; queue draining can stall if jobs are not claimed or retried correctly.

## Validation
- Tests to run:
  - `pytest tests/test_extraction_job_service.py tests/test_watcher.py tests/test_extraction_pipeline.py -q --basetemp=.pytest-tmp-06-02`
- Manual verification:
  - drop a supported file into `ingest/`
  - verify a queued extraction job is created
  - run `python -m src.main process` and confirm package reaches `EXTRACTED`
- Success criteria:
  - watcher no longer runs extraction inline
  - extraction jobs are enqueued once per package
  - failed jobs retry up to the configured limit
