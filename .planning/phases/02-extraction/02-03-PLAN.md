<task type="auto">
  <name>Task 1: Implement Extraction Pipeline with PDF Handling</name>
  <files>src/services/extraction_pipeline.py</files>
  <action>
    Implement `ExtractionPipeline` class:
    - `process_package(package_id: str)`:
      1. Fetch `Package` and its `ExtractedFile` records from the database.
      2. Handle document conversion:
         - For `application/pdf`: Use `pdf2image.convert_from_bytes` to get images for each page.
         - For images: Use `Pillow` to load and get dimensions.
         - For text (email body): Aggregate for context.
      3. Classification: Call `ClassificationService.classify` using a merged context of the first page/image and email body.
      4. Extraction:
         - Load schema from `configs/` based on classified type.
         - For each page (image), call `ExtractionService.extract`.
      5. Post-processing:
         - Use `CoordinateScaler` to scale every triplet's `bbox` from Gemini's [0, 1000] to actual image pixels.
         - Store the image dimensions (`width`, `height`) in the `ExtractedFile` record if not already present.
      6. Persistence:
         - Store results in the `extractions` table.
         - Update `Package.status` to `EXTRACTED`.
    - Ensure robust error handling: if extraction fails, set status to `FAILED`.
  </action>
  <verify>
    <automated>python -c "from src.services.extraction_pipeline import ExtractionPipeline; print('Pipeline loaded')"</automated>
  </verify>
  <done>Extraction pipeline implemented with PDF support and coordinate scaling.</done>
</task>
