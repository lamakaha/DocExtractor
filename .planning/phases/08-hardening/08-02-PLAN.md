# 08-02 Plan: Package-Level Selection and Reconciliation

## Objective
- Replace the remaining heuristic “pick a likely primary document and continue” behavior with a clearer package-level selection and reconciliation strategy across candidate files and supporting context.

## Current State
- Classification already uses bounded package context instead of only the first page.
- Multi-page reconciliation exists once a canonical document has been selected.
- The pipeline still relies on relatively simple heuristics when a package contains multiple plausible source files, supporting attachments, or competing evidence.

## Target State
- Package-level candidate selection is explicit about which files are primary, secondary, supporting, or ignored.
- Extraction and reconciliation can reason about supporting package context without silently collapsing everything into one ad hoc primary-file choice.
- Provenance remains anchored to the canonical PDF while package-level selection decisions are testable and repeatable.

## Implementation Steps
1. Review the current primary-file and package-context selection path in classification and extraction orchestration.
2. Introduce a clearer candidate-selection model for canonicalization and extraction input assembly.
3. Extend reconciliation so supporting package artifacts can inform the package-level result without blurring provenance to the canonical PDF.
4. Add focused tests for mixed-file packages, duplicate/secondary candidate handling, and stable package-level decisions.

## Risks
- Compatibility risk: medium; changing file-selection semantics can alter which document becomes the canonical extraction source.
- Migration risk: low to medium; better package reasoning may expose assumptions in older tests or sample data.
- Operational risk: medium; over-ambitious package merging could make provenance harder to explain if not bounded carefully.

## Validation
- Tests to run:
  - `pytest tests/test_classification.py tests/test_reconciliation_service.py tests/test_extraction_pipeline.py -q --basetemp=.pytest-lite-08-02`
- Manual verification:
  - ingest a package with multiple plausible candidate files
  - confirm the selected canonical document and any supporting artifacts are inspectable and stable across reruns
  - confirm reviewer grounding still points to the canonical PDF pages and not to ambiguous mixed artifacts
- Success criteria:
  - package-level selection is explicit and testable
  - supporting artifacts can influence reconciliation without breaking canonical PDF provenance
  - mixed-file packages no longer depend on fragile implicit primary-file heuristics
