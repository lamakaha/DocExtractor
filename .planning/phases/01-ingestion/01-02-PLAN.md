<objective>
Implement the core ingestion logic for recursive unpacking of EML and ZIP archives using TDD. Extract plain text from email bodies and group all extracted components into a single Package context.

Purpose: Implement the recursive extraction core of DocExtractor (REQ-ING-01, 02, 03).
Output: Working ingestor with recursive capability.
</objective>

<feature>
  <name>Recursive Ingestion Engine</name>
  <files>src/services/ingestor.py, tests/test_ingestor.py</files>
  <behavior>
    The Ingestor should:
    - Accept a byte stream or file path.
    - If EML: Parse using mail-parser, extract body text, then recursively process each attachment.
    - If ZIP: Extract using zipfile, then recursively process each extracted file.
    - Identify file types (MIME) and keep track of original nested paths.
    - Return a list of ExtractedFile objects with content and metadata.

    Cases:
    - Single PDF inside EML: Returns 1 body text + 1 PDF file.
    - ZIP containing (PDF, ZIP containing Image): Returns 1 PDF + 1 Image.
    - EML containing ZIP: Returns 1 body text + files inside ZIP.
  </behavior>
  <implementation>
    - Create a `RecursiveIngestor` class.
    - Use `io.BytesIO` to handle file streams in memory.
    - Implement a `_recursive_extract(content, filename, path_prefix)` method.
    - Use `mail-parser` for body text extraction (favoring plain text over HTML).
    - Map outputs to `ExtractedFile` schema.
  </implementation>
</feature>
